# GATE OWED records (normative home)

This file is the NORMATIVE home for the tests-journal GATE OWED record machinery: the closure append's shape and coverage validity, rename re-binds, lost referents, the machine-local out/ identity, and row retirement. `SKILL.md` keeps the always-hot duty — the session-start `^GATE OWED` grep — and points here.

**Trigger — read this file ENTIRELY, never a fragment:** an open `GATE OWED` line found at session start, a closure append about to be written, or a rename/retirement touching a line's named files.

An owed gate found runs now, and ONLY a run SCORED green is recorded by APPENDING to the owed line — `— gate run <ISO date>, <green out/ dir>, scenarios: <names scored green>` — the scenario list makes the closure's COVERAGE claim readable on every machine (checkable against map.tsv without the machine-local dir); the same append-the-answer shape as the family's other records (a red run never answers the line: it stays open through the refactor); a `GATE OWED` line without that append is mechanically open — and an append whose `scenarios:` list does NOT cover the scenarios map.tsv maps to the line's named files is INVALID: the line stays open (the session-start check verifies the list against map.tsv AS OF the gate run's TIMESTAMP (the out/ dir name carries it — `green-YYYYMMDD-HHMMSS`; the mapping the run could have known); rows added later — including later the same day — create coverage duties for future edits, never re-open a recorded closure (a recorded answer is never re-litigated) — the paired-effects principle: an append closes only when its run did what closure requires, checked by shape, never assumed). Renaming or moving a file an open owed line names re-binds the line IN THE SAME commit — append `— now <new path>` (the record follows its referent); an open owed line whose named files map to ZERO scenarios in map.tsv is a LOST REFERENT — resolved with the user, never read as vacuously satisfied. The cited out/ dir is machine-local (gitignored): it is the run's identity on the machine that ran it — a later reader elsewhere re-verifies by RE-RUNNING the gate, never by demanding the dir; the append itself stays the mechanical closure. Map rows are not append-forever: a scenario proven model-default across cohorts (the journal's insurance-rule class), or fully subsumed by a newer scenario, MAY be retired or consolidated — by a dated journal record naming the row and the evidence, never a silent row deletion; the gate budget is owned, not merely accumulated.
